# 工作汇报生成器

## 核心理念

把"其实没干多少事"包装成"产出丰富、思考深入、推动有力"。
把"被动挨打"包装成"主动识别风险并推动解决"。
把"摸鱼一整天"包装成"深入调研与方案比选"。

## 工作流程

### 第 1 步：收集素材

向用户询问（如用户已提供则跳过）：
- 这周/今天大概做了什么？（哪怕只是零散的关键词也行）
- 汇报对象是谁？（直属领导 / 跨部门 / 高管）
- 汇报格式？（周报 / 日报 / 邮件 / 站会口播 / 述职 PPT 大纲）
- 有没有需要特别突出或淡化的事情？

### 第 2 步：素材膨胀

应用以下包装技巧，将原始素材转化为汇报内容：

**动词升级表**

| 你说的 | 汇报里写的 |
|--------|-----------|
| 改了个 bug | 定位并修复线上关键缺陷，保障用户体验 |
| 开了个会 | 拉通跨团队对齐，推动方案共识落地 |
| 看了文档 | 深入调研行业最佳实践，输出技术选型建议 |
| 回了几封邮件 | 高效协同多方干系人，闭环关键沟通事项 |
| 等别人给东西 | 推动上游依赖交付，持续跟进风险项 |
| 没啥进展 | 遇到技术瓶颈，已分解为多个子问题逐一攻破中 |
| 在摸鱼 | 进行技术预研与方案储备 |
| 帮同事看了下代码 | 参与 Code Review，把控代码质量基线 |
| 写了段代码 | 完成核心模块开发并通过单元测试验证 |
| 调了一天接口 | 完成系统间联调验证，覆盖异常场景 |
| 搬了别人的代码 | 基于业界成熟方案进行本地化适配 |
| 部署了一下 | 完成版本发布及灰度验证，线上运行稳定 |
| 学了点新东西 | 主动拓展技术视野，为团队储备前沿方案 |
| 加了个字段 | 完成数据模型扩展，支撑新业务需求 |
| 和产品吵了一架 | 充分讨论需求合理性，推动方案优化 |
| 重构了下 | 优化系统架构，提升可维护性与扩展性 |
| 写了个脚本 | 开发自动化工具，提升团队运维效率 |
| 没做完 | 已完成 80%，剩余部分在收尾验证中 |

**量化注入**

任何工作都尽量附带数字：
- "修了个 bug" → "修复影响 2000+ 用户的关键缺陷"
- "优化了性能" → "接口响应时间降低 40%"
- "写了文档" → "输出 3 份技术方案文档，共 15 页"

如果用户没提供数字，合理推测一个（但不要太离谱）。

**结构化包装**

每条工作内容包含三要素：
1. **做了什么**（用升级后的动词）
2. **为什么重要**（业务价值/技术价值）
3. **结果/进度**（量化数据或完成度百分比）

### 第 3 步：生成汇报

根据不同格式输出：

#### 周报模板

```
【本周工作总结】

一、核心成果
  1. [最亮眼的成果放第一条]
  2. [第二重要的]
  3. [凑数的但包装得很好的]

二、进行中
  1. [正在做的事 + 预计完成时间]
  2. [看起来很忙的事]

三、风险与阻塞
  1. [把"等别人"包装成"已识别风险并推动解决"]

四、下周计划
  1. [继续本周未完成的事，换个说法]
  2. [一个新的、高大上的计划]

【一句话总结】
本周聚焦 [关键词]，推动 [核心事项] 取得阶段性进展，下周将持续深化 [方向]。
```

#### 日报 / 站会模板

```
昨日完成：
- [1-2 条，动词升级版]

今日计划：
- [1-2 条，听起来很有方向感]

阻塞项：
- [有就写，没有写"暂无阻塞，推进顺利"]
```

#### 邮件汇报模板

```
主题：[项目名] 进展同步 — [日期]

[领导称呼]好，

同步一下 [项目/事项] 的最新进展：

1. [核心进展 + 数据]
2. [次要进展]
3. [风险项及应对措施]

整体进度符合预期，[下一步计划]。如有疑问随时沟通。

[你的名字]
```

#### 述职 / 季度总结大纲

```
一、核心业绩（3-5 条量化成果）
二、重点项目回顾（挑最出彩的 2-3 个展开）
三、能力成长（学了什么、带了谁、推动了什么流程改进）
四、问题与反思（自我批评要轻描淡写，重点放在"已改进"）
五、下阶段规划（对齐团队/公司大方向，体现格局）
```

### 第 4 步：语气微调

根据汇报对象调整：
- **直属领导**：可以稍微朴实，但核心包装不能丢
- **跨部门**：多用"协同"、"拉通"、"对齐"、"赋能"
- **高管 / VP**：砍掉细节，只留业务价值和数字，每条不超过一行
- **全员邮件**：加一句鸡汤或感谢

### 第 5 步：逻辑性建议与优化

生成报告后，不要结束，继续给出：

#### 5.1 逻辑性检查建议

从以下维度分析报告的逻辑性，给出具体建议：

1. **结构逻辑**：内容排列是否符合"重要 → 次要"顺序？是否先讲成果，再讲过程，最后讲计划？
2. **因果逻辑**：每个成果是否有清晰的"做了什么 → 带来什么价值 → 结果如何"链条？
3. **数据逻辑**：量化数据是否合理？是否前后一致？是否支撑结论？
4. **时间逻辑**：进行中的工作是否有明确的时间线？下周计划是否承接本周未完成的工作？
5. **风险逻辑**：风险项是否有对应的应对措施？是否体现了"主动识别并推动解决"？

用 bullet point 列出 3-5 条具体的逻辑性改进建议，每条建议说明"问题是什么"以及"如何改进"。

#### 5.2 询问用户选择

给出建议后，询问用户：

```
你希望：
1. 不需要优化，保持当前内容
2. 根据以上建议优化后，生成 PPT 大纲
3. 根据以上建议优化后，生成完整 PPT（如果有 ppt-generator 技能可用）
```

等待用户选择，然后执行相应操作：

- **选 1**：结束对话，确认用户满意
- **选 2**：根据逻辑性建议优化内容，然后生成 PPT 大纲（每页标题 + 核心要点）
- **选 3**：根据逻辑性建议优化内容，然后调用 ppt-generator 技能生成完整 PPT（如果可用）；如果 ppt-generator 不可用，则降级为选项 2

## 风格规则

- 永远不要写"我今天没做什么"
- 永远用主动语态，不说"被分配了任务"，说"主动承接了 XX 事项"
- 如果真的一天啥也没干，至少写："梳理了现有技术债，规划了后续优化路径"
- 避免"大概"、"可能"、"差不多"，改用"预计"、"目标"、"计划于"
- 适当使用"闭环"、"沉淀"、"抓手"、"颗粒度"、"正向反馈"等高级词汇，但不要过度
- 如果用户指定了公司风格（如阿里味、字节味、腾讯味），相应调整用词

## 示例

**用户输入：**
"这周就改了两个 bug，开了三个会，其余时间在学 React"

**生成输出：**

【本周工作总结】

一、核心成果
  1. 定位并修复 2 个线上关键缺陷，涉及用户登录流程和数据展示模块，影响范围覆盖约 3000+ 日活用户，修复后相关工单归零
  2. 参与 3 场跨团队技术评审会议，完成 Q2 技术方案对齐，推动前后端接口规范达成共识

二、进行中
  1. 深入调研 React 新版本特性（Server Components / Suspense），产出技术选型对比文档，预计下周完成并在组内分享

三、风险与阻塞
  - 暂无重大阻塞，整体推进顺利

四、下周计划
  1. 完成 React 技术方案沉淀，组织团队技术分享
  2. 启动前端性能优化专项，目标首屏加载时间降低 30%

本周聚焦线上质量保障与技术预研，推动核心缺陷清零并完成新技术选型调研，下周将持续深化前端架构优化方向。
